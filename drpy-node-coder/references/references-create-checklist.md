# drpy-node Source Create 专项参考

时间：2026-04-18 14:27 CST
用途：专门服务于“新建源”阶段，帮助判断模板继承、分类入口、搜索入口、是否该最小覆盖。

---

## 1. 新建源阶段必须先回答的 5 个问题

### Q1. 当前站更像哪个模板？
优先比对：
- `mx`
- `mxpro`
- `首图`
- `首图2`

### Q2. 当前模板默认 `class_parse` 是什么？
如果准备使用：
```js
class_name
class_url
```
必须先检查模板是否残留 `class_parse`。

### Q3. 当前模板默认 `double` 是什么？
很多模板默认：
```js
double: true
```
如果首页推荐实际是单层结构，需要重点怀疑这个点。

### Q4. 当前站真实分类 `url` 是什么？
不能只看首页导航表面链接，必须验证：
- 分类第一页
- 分类第二页
- 翻页结构
- `fyclass/fypage` 落位

### Q5. 当前站真实搜索 `searchUrl` 是什么？
搜索模板不能拍脑袋写，必须看：
- 搜索第一页
- 搜索第二页（如果有）
- 是否命中站内搜索页真实容器

### Q6. 源文件 metadata 是否自洽？
新建源写入前必须确认：
- `@header.title` / rule `title` / 文件名表达的是同一个源。
- `@header` 的 `类型` 与实际内容一致（影视/漫画/小说/听书等）。
- `lang: 'ds'`、`searchable`、`filterable`、`quickSearch` 与源实际能力一致。
- 特殊内容源已明确预期协议（`pics://`、`novel://`、音频直链、`push://`）。
- 仓库 tags 不在 create 阶段脑补，上传标签交给 repo-upload。

## 2. 模板继承后优先检查表

| 字段 | 为什么必须先看 |
|---|---|
| `class_parse` | 可能压住 `class_name/class_url` |
| `double` | 可能导致首页推荐为空 |
| `url` | 决定一级是否能通 |
| `searchUrl` | 决定搜索是否能通 |
| 默认 `一级` | 可能已经够用，不必手写 |
| 默认 `搜索` | 可能已经够用，不必手写 |

---

## 3. 新建源阶段的强纪律

### 纪律 A
先确认模板默认定义，再决定要不要覆盖。

### 纪律 B
先确认真实 `url/searchUrl`，再讨论选择器。

### 纪律 C
如果模板内置规则可能够用，先不要写 `一级/搜索`。

### 纪律 D
如果自动评估没串起来，不要先判源死；先拆接口验证。

### 纪律 E
源内 metadata 只描述运行时源身份和能力，不代表仓库 tags；如果用户要求上传或改标签，交给 repo-upload。
