# 继承模板站最小覆盖开发经验总结

> 适用于这类 **已命中内置模板，且通过模板继承生成最终 rule，但仍需要最小覆盖修正首页推荐 / 一级 / 二级 / 搜索的继承模板站**。

---

## 一、先判断是否属于“继承模板站”问题，而不是具体模板个案

### 核心原则
这类经验不应被限制为：
- 只适用于 `mx`
- 只适用于某个具体模板名

而应抽象为：

**只要站点是通过模板继承得到最终 rule，并出现首页推荐空、detail 不通、搜索为空等问题，都优先按继承模板站思路排查。**

### 经验
模板名（如 `mx`、`mxpro` 等）只是入口，不是经验边界。
真正有复用价值的是：
- 模板继承后的最终字段如何影响行为
- 哪些问题优先查模板继承参数而不是重写全规则

---

## 二、继承模板站首页推荐空时，优先检查 `double`

### 通用结论
当继承模板站满足以下条件：
1. 推荐列表节点本身已确认存在
2. 手写 `推荐` 已直接落到最终卡片层
3. `home.class` 正常但 `home.list` 为空

必须优先怀疑：

```js
double: true
```

### 原因
继承模板常会对首页推荐做双层处理；
若站点实际是单层推荐结构，则会把原本存在的推荐节点处理空。

### 实战原则
对于单层推荐结构，应优先尝试：

```js
double: false
```

### 结论提升
这不是某个模板个案经验，而是：

**所有继承模板站在“首页推荐空但推荐节点真实存在”时，都应优先检查 `double`。**

---

## 三、列表定位要分首页、分类、搜索三套结构分别判断

### 通用原则
继承模板站中：
- 首页推荐
- 一级分类页
- 搜索结果页

很可能不是同一套卡片结构。

### 经验
不要把分类页的列表定位直接套到首页或搜索页，也不要因为命中模板就默认它们结构一致。

### 实战原则
必须分别确认：
- 首页推荐列表容器
- 一级分类列表容器
- 搜索结果列表容器

---

## 四、detail 不通先查 `detailUrl`，不要先怀疑二级字典

### 通用现象
当二级字典看起来合理，但 detail 传纯数字 ID时：
- 返回模板兜底值
- 看起来像二级规则完全没生效

### 通用优先检查项
首先检查：

```js
detailUrl
```

### 原因
继承模板站 detail 串联常依赖：
- 一级返回完整详情链接
- 或者由 `detailUrl` 把纯数字 id 转成详情页 URL

若缺失 `detailUrl`，即使二级字典完全正确，也会表现成“detail 进不去”。

### 实战原则
**所有继承模板站在 detail 不通时，先查 `detailUrl`，再查二级字典。**

---

## 五、二级能用字典解决时，不要急着上 async

### 通用结论
如果站点详情结构仍是传统模板站形态：
- 标题可定位
- 图片可定位
- tabs/lists 可定位

优先尝试：
- 二级字典最小覆盖

而不是直接：

```js
二级: async function () {}
```

### 实战原则
继承模板站能用字典解决时，先保留模板体系，避免无谓扩大覆盖范围。

---

## 六、搜索为空时，优先检查搜索页是否是独立 DOM

### 通用现象
继承模板站搜索结果为空时，经常不是 `searchUrl` 完全错误，而是：
- 搜索页结构与一级完全不同
- `搜索: '*'` 或默认继承一级结构失效

### 实战原则
搜索结果为空时优先检查：
1. `searchUrl` 是否为真实搜索翻页模板
2. 搜索页是否使用独立 DOM（如 `.searchlist_item`）
3. 是否需要单独写 `搜索`，而不是继续继承一级

### 结论提升
这同样不应被写成某个模板个案，而是：

**所有继承模板站在搜索为空时，都应优先判断“搜索页是否独立于一级结构”。**

---

## 七、播放不强求直链，通用 lazy 没收益时接受原页面嗅探

### 通用经验
若验证发现：
- 不写自定义 `lazy`
- 与写 `lazy: 'js: return input;'`

最终都只得到：

```js
{ parse: 1, jx: 0, url: vodplay页 }
```

说明引擎通用处理不会自动提升成更优直链。

### 实战原则
这类情况下，不必强行深挖播放；
若站点能正常走原页面嗅探，就接受它。

---

## 八、继承模板站最小覆盖的正确打开方式

### 通用原则
继承模板站并不意味着完全不能写覆盖，
但覆盖应遵守：

**只覆盖真正失效的链路，不要把整个模板体系推翻重写。**

### 常见最小覆盖项
- `url`
- `searchUrl`
- `detailUrl`
- `double`
- `推荐`
- `一级`
- `二级`（字典最小覆盖）
- `搜索`（若搜索页结构独立）

---

## 九、继承模板站通用排障顺序

推荐顺序：

1. 先确认是否属于继承模板站问题
2. 先确认 `url` 是否为真实分类翻页模板
3. 首页推荐空时优先检查 `double`
4. 一级列表单独确认定位
5. detail 不通优先检查 `detailUrl`
6. 搜索为空优先检查搜索页 DOM 是否独立于一级
7. 播放若引擎通用 lazy 无额外收益，则接受原页面嗅探

---

## 十、最重要的教训

### 不要把经验写窄成某个模板名的个案
这次真正有价值的不是“mx 怎么修”，而是：

- 继承模板站首页推荐空 → 优先查 `double`
- 继承模板站 detail 不通 → 优先查 `detailUrl`
- 继承模板站搜索空 → 优先查搜索页是否独立 DOM

### 经验提升后的通用结论
**模板名只是入口，继承后的 rule 行为和最小覆盖策略，才是可复用的方法论。**
